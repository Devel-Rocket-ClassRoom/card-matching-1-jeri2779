## 게임 규칙

- 4x4 격자(총 16장)에 1~8까지의 숫자가 **2장씩** 뒤집어져 있습니다.
- 게임 시작 시 카드를 무작위로 섞습니다.
- 매 턴마다 **행과 열 번호**를 입력하여 카드 **2장**을 선택하고 뒤집습니다.
- 두 카드의 숫자가 **같으면** 짝이 맞은 것이며, 카드가 공개된 상태로 유지됩니다.
- 두 카드의 숫자가 **다르면** 다시 뒤집힙니다.
- **8쌍을 모두 찾으면** 게임 클리어입니다.

## 작업 순서
- 1~8까지의 숫자 2개씩 있는 카드 클래스
- 카드를 놓을 보드 클래스(배열 구현)
- 카드를 Shuffle하는 메서드
- 카드를 배열에 배치 
- 카드가 배치된 보드 상태를 출력하는 메서드
- 카드가 배치된 보드 상태는 **로 표시되는 Hidden 상태로 시작
- 
- 배치 방법은 shuffle한 카드에서 하나씩 순차적으로 보드에 배치
- 배치된 카드는 ** 로 표시되어야 함.
- 시도횟수/ 찾은 쌍 수를 저장하는 변수와 출력을 위한 메서드
- 사용자 입력 받아서 카드를 공개하는 메서드(행과 열 입력받아서 카드 공개)
- 입력은 행 공백 열 형태로 받음(예외 처리 필요)
- 2번의 입력후 카드의 숫자가 일치하는지 비교하는 메서드
- 일치할시 찾은 쌍 count증가, 시도 횟수 증가, 카드 공개 상태 유지
- 불일치할시 시도 횟수 증가, 카드 다시 뒤집기
- 게임 종료시 시도 횟수와 찾은 쌍 수 출력, 새 게임 여부 입력받아서 Y면 새 게임 시작, N이면 프로그램 종료

## 질문 사항
- 카드 클래스의 카드는 동일한 숫자 2개씩 생성후 shuffle해서 저장. 생성방식은 for문 사용?
- 보드 배열 구현은 현재 기본 4x4로 가정하고 2차원 배열로 구현?
- 행과 열을 입력받는 메서드는 어떻게 처리해야할지? 예외처리는 어떻게 해야할지? readline으로 입력후 정수변환해서 해당 배열 인덱스 접근방식?
- 기본 보드 상태는 **로 표시되는 Hidden 상태로 시작하는데 플레이어가 입력한 행과 열의 카드 공개 방식은?
- 일치 여부 비교는 공개된 카드(해당 배열 인덱스값)의 숫자를 == 연산자로 비교하는 방식?
- 불일치 할때 카드를 다시 Hidden 상태로 변경하는 별도 메서드 필요?
- Thread.Sleep()이 요구사항에 있으므로 불일치시와 카드 섞을때 일정 시간 대기할때 사용?

## 각 단계별 설계 노트
- 카드 클래스는 카드를 생성하고 숫자를 저장한다. 이때 동일한 숫자가 2개씩 있어야 하므로 해당 과정을 2번 반복하는 이중 for문을 사용하여 카드 객체를 생성한다.
- 보드 클래스는 2차원 배열로 구현.[4,4] 이중 for문을 사용하여 구현. 셔플된 카드 덱에서 순차 저장하는 방식으로 보드 배치하고 기본 ** 표기로 구현.
- 사용자 입력은 readlind함수 사용 배열 인덱스와 일치해야 하므로 -1 처리 하고 readline으로 받은 숫자는 정수 변환해서 처리.
- 생성된 카드를 보드 배열에 저장하는 방법은 셔플된 카드 덱에서 하나씩 순차적으로 보드에 배치하는 방식으로 구현.

## 추가 도전 (선택)

기본 게임을 완성했다면 다음 기능을 추가하세요.

### 난이도 선택

게임 시작 시 보드 크기를 선택할 수 있습니다.

| 난이도 | 보드 크기 | 카드 쌍 |
|--------|----------|--------|
| 쉬움 | 2x4 (8장) | 4쌍 |
| 보통 | 4x4 (16장) | 8쌍 |
| 어려움 | 4x6 (24장) | 12쌍 |

**실행 예시:**

```
=== 카드 짝 맞추기 게임 ===

난이도를 선택하세요:
1. 쉬움 (2x4)
2. 보통 (4x4)
3. 어려움 (4x6)
선택: 1

카드를 섞는 중...

    1열 2열 3열 4열
1행  **  **  **  **
2행  **  **  **  **

시도 횟수: 0 | 찾은 쌍: 0/4
```

난이도별로 시도 횟수가 제한됩니다. 제한 횟수 내에 모든 쌍을 찾지 못하면 게임 오버입니다.

| 난이도 | 시도 제한 |
|--------|----------|
| 쉬움 | 10회 |
| 보통 | 20회 |
| 어려움 | 30회 |
